Use the following criteria to evaluate your mock interviews. The more you practice, the better you will do on these criteria.

## Problem solving

### Understanding the problem

ğŸ‘ Understood the key aspects of the problem quickly

ğŸ‘ Had difficulty in understanding the key aspects of the problem

### Solution / Approach

ğŸ‘ Approached the problem in a systematic and logical manner

ğŸ‘ Did not demonstrate a logical thought process for approaching the problem

### Improving the Solution

ğŸ‘ Suggested a more efficient solution when prompted, or proactively coming up with a better solution

ğŸ‘ Had difficulty in coming up with a more efficient solution even after being prompted

### Trade-offs Analysis

ğŸ‘ Explained the trade-offs of different approaches clearly and correctly

ğŸ‘ Failed to describe trade-offs of different approaches

### Hinting

ğŸ‘ Did not require any major hints

ğŸ‘ Needed plenty of hints

## Technical competency

### Speed

ğŸ‘ Quickly implemented a working solution

ğŸ‘ Was not able to complete the solution

### Correctness / Accuracy

ğŸ‘ Implemented the solution correctly with minimal bugs

ğŸ‘ Unable to correctly implement a solution 

### Complexity Analysis

ğŸ‘ Able to determine the algorithmic time and space complexity

ğŸ‘ Was not able to determine the algorithmic time and space complexity 

### Mastery of Chosen Programming Language

ğŸ‘ Demonstrated mastery of the chosen programming language

ğŸ‘ Does not seem to be familiar with the chosen programming language

### Implementation

ğŸ‘ Implementation was clean and straightforward

ğŸ‘ Implementation was unnecessarily complex and/or messy

### Coding Style

ğŸ‘ Coding style was neat (proper indentation, spacing and no bad practices)

ğŸ‘ Coding style was messy (inconsistent indentation, weird spacings, etc)

## Testing
### Common Cases

ğŸ‘ Tested their code against various typical cases

ğŸ‘ Failed to test the code against typical cases

### Corner Cases

ğŸ‘ Found and handled corner/edge cases

ğŸ‘ Failed to consider corner/edge cases

### Self-correction

ğŸ‘ Identified and corrected bugs in the code (where applicable)

ğŸ‘ Was not able to discover and fix bugs even after being prompted

## Communication

### Clarify Problem

ğŸ‘ Appropriately asked good, clarifying questions about the problem

ğŸ‘ Failed to confirm understanding/ask appropriate questions

### Communicating Approach

ğŸ‘ Able to explain overall approach, technical terms and acronyms (where applicable)

ğŸ‘ Failed to effectively explain overall approach, technical terms and acronyms (where applicable)
